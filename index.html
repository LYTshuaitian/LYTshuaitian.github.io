Maybe is preface & 我是如何搭建我的Github主页的

发表于 2017-10-25

　　这算是我正式意义上发的第一篇文章，就讲一下我是怎么搭建Github主页，当做自己的第一篇博客的吧。
　　因为一直想搭建一个自己的博客，主要是为了记录一些心得，还有工作中写代码的时候解决的一些问题，记下来以便于自己日后翻阅，其次是一些公司把拥有自己的博客自己的代码库作为招人的参考标准之一，所以为了自己更好的未来克服了自己的懒，结果在写多了博客之后喜欢上了这种感觉，不过这是后话了。
　　在准备搭建之前翻阅了很多资料，国内现在用wordpress的很多，包括同事也用的这个，但是年轻的我想要高大上想要与众不同，简单说就是想要装~，所以看了很多最终决定用Github的个人博客，可是写在这里的博客不会被百度爬到，所以以后也会买一个虚拟主机买个域名的搭建一个国内的，不过眼下先完善这个博客吧。
　　在动手搭建之前看了大概一早上的各种攻略，基本都看了好几遍才开始动手，在心里整理了下大概的步骤，感觉这样比较有用。但是百度到的攻略基本都大同小异，很有互相借鉴抄袭之嫌疑，看多了用处不大，在这里介绍我搭建博客用到的三篇攻略

hexo从零开始到搭建完整
GitHub Pages + Hexo搭建博客
史上最详细“截图”搭建Hexo博客并部署到Github

　　类似这种的攻略一找一大把，具体步骤这里不做介绍，只讲一下我个人搭建遇到的一些问题疑难点等。
　　所有的攻略都是用的ssh和Github仓库建立连接的，用Gitbash敲命令，而我个人用gitgui比较多，因为最开始用git就用的gui，gui对新手比较简单容易学会，敲命令也是在后来我做linux相关操作才喜欢上的，敲命令确实有一种快感（也许是我个人的想法）。说到ssh就有一个问题，所有的攻略都是只讲了创建，对于已经有了ssh怎么办没有人讲，甚至我看到有篇攻略说有了就删掉，大哥如果你别的项目绑定了这个ssh你会把他删掉？你别的项目是不想做了？SSH的文件位置可以用命令寻找
$ cd ~/.ssh,也可以在文件地址里写C:\Users\LYT，LYT是我电脑的名字，各自电脑不同会不一样，在这个目录下有一个.ssh的文件，里面就是你的公匙之类的啦。

千万记得拷贝id_rsa.pub下面的，这是公匙，别拷成私匙了。
